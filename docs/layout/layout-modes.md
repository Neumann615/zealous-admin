# 布局模式

zealous-admin 提供了 5 种布局模式，通过 `menu.menuType` 配置切换。每种模式在导航结构、空间利用率和适用场景上各有不同。

## 配置方式

```tsx
const config: LayoutConfig = {
  menu: {
    menuType: 'side', // 切换布局模式
  },
}
```

## 5 种模式详解

### 1. side（侧边栏模式）

**默认布局**，左侧垂直导航栏 + 顶部 Header。

```
┌──────────┬─────────────────────────────┐
│          │  Header（标签栏 + 工具栏）     │
│  MainNav │─────────────────────────────│
│  （图标） │                             │
│          │                             │
│  Menu    │  Content（内容区）            │
│  （文字） │                             │
│          │                             │
└──────────┴─────────────────────────────┘
```

- 左侧 MainNav 显示一级菜单图标
- Menu 展开显示当前一级菜单下的完整子菜单
- 适合菜单层级较深的项目

### 2. only-side（纯侧边模式）

简化版侧边栏，MainNav 直接承载子菜单，不再单独展开 Menu 面板。

```
┌──────┬─────────────────────────────────┐
│      │  Header（标签栏 + 工具栏）         │
│MainNav│─────────────────────────────────│
│(+子菜单)│                                │
│      │  Content（内容区）                 │
│      │                                 │
└──────┴─────────────────────────────────┘
```

- MainNav 直接渲染所有菜单，支持子菜单弹出
- 节省横向空间
- 适合一级菜单较少的项目

### 3. head（顶部导航模式）

顶部导航栏 + 左侧菜单的混合模式。

```
┌─────────────────────────────────────────┐
│  MainNav（顶部横向导航）                   │
├──────────┬──────────────────────────────┤
│          │  Header                      │
│  Menu    │──────────────────────────────│
│          │                              │
│          │  Content（内容区）             │
│          │                              │
└──────────┴──────────────────────────────┘
```

- MainNav 在顶部横向排列
- 适合需要最大化内容区宽度的场景

### 4. only-head（纯顶部模式）

仅顶部导航，无侧边菜单。

```
┌─────────────────────────────────────────┐
│  MainNav（顶部导航，含子菜单弹出）          │
├─────────────────────────────────────────┤
│  Header                                 │
├─────────────────────────────────────────┤
│                                         │
│  Content（内容区）                        │
│                                         │
└─────────────────────────────────────────┘
```

- 所有菜单在顶部，子菜单以下拉形式展开
- 内容区全宽
- 适合菜单层级简单、内容密集型应用

### 5. simple（精简模式）

最简化的布局，仅保留菜单和内容。

```
┌────────┬────────────────────────────────┐
│        │  Header                        │
│  Menu  │────────────────────────────────│
│        │                                │
│        │  Content（内容区）               │
│        │                                │
└────────┴────────────────────────────────┘
```

- 无 MainNav，直接显示 Menu
- 最简洁的布局
- 适合功能单一的后台应用

## 搭配内容区布局

通过 `app.layout` 选项可进一步调整内容区域：

| 选项 | 效果 |
|------|------|
| `isCenter: true` | 内容区居中显示（适合大屏） |
| `layoutScope: 'inside'` | 布局仅影响内容区内部 |
| `width: 1200` | 固定内容区最大宽度 |
